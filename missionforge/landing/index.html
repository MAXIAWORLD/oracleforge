MissionForge turns a YAML file into a fully autonomous agent pipeline. Chain LLM calls, RAG retrieval, webhooks, cron schedules, and memory storage — then expose everything via REST or MCP.

Get started API reference →
missions/daily-brief.yaml 
name: daily-brief
description: Morning intelligence report
schedule: "0 8 * * *"   # every day 08:00

agent:
  llm_tier: standard

steps:
  - action: rag_query
    query_template: "latest updates"
    max_chunks: 5

  - action: llm_call
    prompt_template: |
      Summarize for a busy exec:
      {{rag_context}}

  - action: webhook
    url: "https://hooks.slack.com/..."
    body_template: "{{llm_output}}"

01

Everything wired together

One framework, zero glue code. Each feature is a YAML key away.

📄

YAML missions

Define agents, steps, schedules and LLM tiers in a single human-readable file. Version it in git.

🧠

Multi-tier LLM routing

Auto, standard, and premium tiers route calls to the right model automatically — Anthropic, local, or any OpenAI-compatible endpoint.

🔍

RAG built-in

Ingest documents via the REST API. ChromaDB vector store is wired in by default — query context flows straight into LLM prompts.

⏱

Cron scheduling

Add a cron expression and MissionForge runs the mission on schedule — no Celery, no Redis required.

🔗

Webhooks & HTTP steps

POST results to Slack, Notion, Zapier or any endpoint. Full response available as template variable for the next step.

💾

Run history

Every execution is persisted in SQLite (or Postgres). Query by mission, status, or date range via REST.

🤖

MCP server

5 tools exposed via SSE and Streamable HTTP transports — plug into Claude Desktop, Cursor, Zed, or Glama instantly.

🔑

API key auth + rate limiting

Production-ready middleware out of the box — key auth, per-IP rate limiting, and security headers on every endpoint.

02

Up in 60 seconds

Self-host on any machine that runs Python 3.12.

  1. 01

    Install

    terminal 
    pip install missionforge
  2. 02

    Configure

    Create a .env file with your API key and database URL.

    .env 
    SECRET_KEY=your-secret-key-32-chars
ANTHROPIC_API_KEY=sk-ant-...
DATABASE_URL=sqlite+aiosqlite:///missionforge.db
MISSIONS_DIR=./missions
  3. 03

    Write your first mission

    missions/hello.yaml 
    name: hello
steps:
  - action: llm_call
    prompt_template: "Say hello from MissionForge!"
  4. 04

    Start the server and run

    terminal 
    # Start server
uvicorn missionforge.main:app --port 8001

# Run mission via REST
curl -X POST http://localhost:8001/missions/hello/run \
  -H "X-API-Key: your-secret-key-32-chars"
