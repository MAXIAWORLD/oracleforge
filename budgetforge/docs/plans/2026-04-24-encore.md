# Plan "encore" — Audit QA Senior BudgetForge

Date : 2026-04-24
Contexte : proxy LLM avec enforcement budget. Stack FastAPI + SQLite + slowapi + Next.js. 2 workers uvicorn. Pas de Redis. URL prod : https://llmbudget.maxiaworld.app

## Tableau des risques

| Gravité | Phase | Composant | Problème |
|---|---|---|---|
| **CRITIQUE** | 1 | `distributed_budget_lock.py` + 2 workers uvicorn sans Redis | Fallback `asyncio.Lock` par-worker. Worker A et B peuvent check budget simultanément → overshoot garanti sous charge. |
| **CRITIQUE** | 2 | `core/auth.py` + `config.py` | `admin_api_key=""` par défaut = dev mode public. `.env` oublié en prod → `POST/PUT/DELETE /api/projects/*`, `/api/admin/stats`, `rotate-key` ouverts à tous sans auth. Aucun fail-fast quand `app_env="production"`. |
| **CRITIQUE** | 1 | `dynamic_pricing.py` | Prix de facturation/budget pilotés par APIs tierces (OpenRouter, Together) **sans validation**. Si OpenRouter retourne `prompt=0` pour un modèle → budget bypass complet. Cache 5 min empoisonne tous les projets. |
| **HAUT** | 2 | `core/url_validator.py` + `send_webhook` | SSRF TOCTOU : DNS résolu à la validation, re-résolu à l'envoi. TTL 0 côté attaquant → première résolution publique passe, seconde pointe 169.254.169.254 (metadata AWS). Pas de pinning IP ni `local_addr` restreint. |
| **HAUT** | 1 | `token_estimator.py` + `check_per_call_cap` | `estimate_output_tokens` = `max_tokens` si présent, sinon heuristique (×0.75). Client omet `max_tokens` sur un prompt de 100 tokens → estimate=75, réel=8000, cap implicite inutile, **prebill sous-facture, finalize arrive après overshoot**. |
| **HAUT** | 2 | `routes/billing.py` `reconcile_stripe_session` | `GET /api/billing/reconcile/{session_id}` **sans auth, sans rate-limit**. Un `session_id` valide crée un Project arbitraire. Pas idempotent si `subscription_id` absent (session `free`). |
| **HAUT** | 1 | `proxy_forwarder.forward_azure_openai` | Signature `(body, key, base_url, timeout)` mais `dispatch_openai_format` appelle `(body, key, timeout_s=...)`. TypeError au runtime. Provider inutilisable — jamais testé en prod. |
| **HAUT** | 2 | `plan_quota.check_project_quota` | Filtre `Project.name == owner_email`. Si admin crée un Project avec `name="staging"` → n'est pas compté dans le quota free. **Plan free (1 projet) bypassable** via l'API admin. |
| **HAUT** | 1 | `prebill_usage` + finalize | `db.commit()` non protégé contre `OperationalError: database is locked`. SQLite 30s busy_timeout mitigate, pas élimine. Si cancel_usage échoue post-4xx provider → prebill fantôme → budget inflaté définitivement. |
| **MOYEN** | 3 | `lib/api.ts` | `X-Admin-Key` dans localStorage (XSS = key volée). Aucune UI pour set/reset la clé — l'utilisateur doit taper `localStorage.setItem` manuellement. |
| MOYEN | 1 | `Usage.cost_usd` Column Float | Décision d'audit passée disait Decimal. Toujours Float. Erreur d'arrondi cumulée sur volumes élevés. |
| MOYEN | 2 | `portal.py` `_verify_session` | Cookie HMAC signé, TTL 90j, **non révocable**. Fuite portal_secret = session takeover massive sans rotation possible. |
| MOYEN | 2 | `signup.py` domain limit 10/jour + IP 3/jour | 10 domaines × 3 IPs = 30 comptes free/jour sans captcha (Turnstile optionnel en dev). Pas de protection contre email+tag (`a+1@gmail.com`, `a+2@gmail.com` = même mailbox, domaines identiques mais Turnstile permissif). |
| MOYEN | 1 | `dispatch_*` streaming | Tokens lus depuis le SSE du provider. Client qui ferme la connexion avant `message_delta` → `got_usage=False` → `cancel_usage` → **appel gratuit** (provider a déjà facturé côté upstream). |
| MOYEN | 3 | Dashboard `/projects/[id]` | `allowedProviders` état local — si backend update fail mais localStorage refresh ne s'aligne pas, UI diverge du DB. Pas de refetch forcé sur erreur 500. |
| BAS | 2 | `limiter.py` | Compteurs in-memory per-worker → limite effective = limit × 2 workers. Non-documenté. |
| BAS | 3 | Mobile <375px | Tables `Agent Breakdown`, `Provider Breakdown` non testés. Pas de `overflow-x-auto` sur certaines listes. |

## Ce qui est solide

- **Stripe webhook** : signature vérifiée via `construct_event`, idempotence `StripeEvent.event_id` avec catch `IntegrityError` (course propre).
- **Admin-key comparison** : `hmac.compare_digest` partout — pas de timing attack sur la clé.
- **Grace period rotate-key** : `previous_api_key + key_rotated_at` avec fenêtre 5 min, code propre.
- **Signup captcha path** : Turnstile intégré, pass-through seulement en dev, warning loggué en prod.
- **prebill/finalize/cancel** : séquence correcte pour race budget mono-worker.

## Correctifs prioritaires

### CRITIQUE — Race condition budget multi-worker
`distributed_budget_lock.py` — fallback asyncio.Lock n'est partagé qu'intra-process. Avec 2 workers, aucune sérialisation. Installer Redis (requis, pas optionnel) OU passer en single-worker (`uvicorn --workers 1`) OU implémenter un advisory lock SQLite (`BEGIN IMMEDIATE` explicite autour de `check + prebill`). Supprimer le fallback silencieux — faire fail loud au démarrage si Redis indisponible en prod.

### CRITIQUE — Dev mode ouvert en prod
`core/auth.py` + `config.py` — Ajouter dans `Settings` un validator : si `app_env=="production"` et `admin_api_key==""` → raise au startup. Idem `portal_secret`, `stripe_webhook_secret`. Aujourd'hui l'app démarre joyeusement sans auth.

### CRITIQUE — Prix pilotés par APIs tierces
`dynamic_pricing.py` — Inverser la priorité : prix statiques `_PRICES` font autorité, dynamic seulement pour modèles nouveaux. Valider fourchette (`0 < input_per_1m < 10_000`) avant cache. Ajouter signature/allowlist des sources.

### HAUT — SSRF TOCTOU webhook
`core/url_validator.py` + services webhook — Résoudre DNS une seule fois, pinner l'IP résolue dans la requête httpx (`transport` custom), OU utiliser une sortie réseau restreinte au niveau OS (namespace réseau sans route vers RFC1918/link-local).

### HAUT — Overshoot via estimate_output_tokens
`token_estimator.py` + `check_per_call_cap` — Si `max_tokens` absent, imposer une valeur minimale de sécurité (8000 ou modèle-context-max). Le cap implicite doit utiliser la borne pessimiste, pas 0.75×input.

### HAUT — `/api/billing/reconcile` ouvert
`billing.py` — Ajouter rate-limit + vérifier que le `session_id` vient bien de Stripe (fetch puis check `livemode`, `payment_status`). Idempotence explicite sur `session.id`, pas juste `subscription_id`.

### HAUT — Azure OpenAI cassé
`proxy_forwarder.forward_azure_openai` — Route `/proxy/azure-openai/*` plante au premier appel. Soit supprimer la route, soit injecter `settings.azure_openai_base_url` dans `dispatch_openai_format`.

### HAUT — Bypass quota free via name≠email
`plan_quota.check_project_quota` — Filtrer sur `Project.owner_email` (ajouter la colonne, migrer) plutôt que `Project.name`. Ou ajouter une constraint qui lie `name` à l'email du créateur pour les plans free.

### HAUT — Cancel/finalize commit non résilient
`proxy_dispatcher.prebill_usage/cancel_usage/finalize_usage` — Try/except sur `db.commit()`, rollback + retry 1× en cas d'`OperationalError`. Logger en critique si échec — sinon le budget dérive silencieusement.
