## 1. Design rationale

### Why server-side history?

Upstream sources (Pyth, Chainlink, RedStone) expose live prices only.
An AI agent that needs to reason about trends, compare current price to
24h ago, or detect slow-moving anomalies has no data to work with unless
the oracle records snapshots over time. V1.8 adds a lightweight
server-side history layer that piggybacks on the existing Pyth Hermes
batch endpoint — one HTTP call per cycle covers ~79 symbols.

### Why SQLite?

The price_snapshots table is append-mostly, write-once-read-many. SQLite
handles this well with minimal operational overhead (no external DB to
manage). A single `(symbol, sampled_at)` composite index covers all query
patterns. The 30-day retention window with auto-purge keeps the DB small
(~79 symbols x 288 samples/day x 30 days = ~680k rows at steady state).

### Downsampling strategy

Raw snapshots are stored at 5-min granularity. Wider intervals (1h, 1d)
are computed at query time via SQL `GROUP BY (sampled_at / bucket_s) * bucket_s`
with `AVG(price)`. This avoids maintaining multiple materialized views
while keeping queries fast (the index makes range scans cheap, and the
2000-datapoint cap bounds result size).

## 2. Schema

```sql
CREATE TABLE IF NOT EXISTS price_snapshots (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol       TEXT NOT NULL,
    price        REAL NOT NULL,
    source_count INTEGER NOT NULL DEFAULT 1,
    sampled_at   INTEGER NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_snapshots_lookup
    ON price_snapshots (symbol, sampled_at);
```

## 3. Background sampler

- Runs as an `asyncio.Task` started in the FastAPI lifespan.
- Calls `price_cascade.get_batch_prices(ALL_FEEDS)` — single Pyth Hermes
  batch call for ~79 symbols.
- Filters out zero-price results, inserts valid rows via
  `insert_price_snapshots()`.
- Every 288 cycles (~24h), purges rows older than
  `HISTORY_RETENTION_DAYS` (default 30).
- Configurable via env vars: `HISTORY_SAMPLE_INTERVAL_S` (default 300),
  `HISTORY_RETENTION_DAYS` (default 30).

## 4. REST endpoint

```
GET /api/price/{symbol}/history?range=24h&interval=5m
```

| Param    | Values           | Default | Notes                         |
|----------|------------------|---------|-------------------------------|
| range    | 24h, 7d, 30d    | 24h     | Time window                   |
| interval | 5m, 1h, 1d      | auto    | 24h->5m, 7d->1h, 30d->1d     |

- Requires API key (X-API-Key header).
- Symbol is case-insensitive (uppercased server-side).
- Returns `{ data: { symbol, range, interval, count, datapoints }, disclaimer }`.
- Each datapoint: `{ timestamp, price, samples }`.
- Max 2000 datapoints per response.
