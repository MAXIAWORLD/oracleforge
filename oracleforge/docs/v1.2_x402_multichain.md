# MAXIA Oracle — V1.2 : x402 micropaiements multi-chain

**Status** : plan figé (pré-implémentation).
**Target** : accepter les paiements x402 sur **Base + Arbitrum + Optimism + Polygon** (4 chaînes EVM, USDC natif partout).
**Effort** : ~1 jour.
**Breaking changes** : aucun (rétrocompat stricte — un client V1.0 qui envoie un tx sur Base sans déclarer la chaîne continue de fonctionner).

---

## 1. Décisions figées

| # | Sujet | Décision | Motivation |
|---|---|---|---|
| D1 | Treasury address | Même adresse EVM (un seul keypair) reçoit USDC sur les 4 chaînes. Fallback cascade : per-chain env override → `X402_TREASURY_ADDRESS` générique → `X402_TREASURY_ADDRESS_BASE` (rétrocompat V1.0). | Une même adresse EVM contrôle 4 réseaux avec la même clé. Obliger 4 wallets séparés double le risque opérationnel sans gain réel. |
| D2 | Facilitator | Base : Coinbase facilitator conservé (officiel). Arbitrum/Optimism/Polygon : **direct on-chain uniquement** (pas de facilitator tiers). | Coinbase n'expose pas de facilitator multichain en 2026-04. Direct RPC = zéro dep externe, plus robuste. |
| D3 | Header de chaîne | `X-Payment-Network: base\|arbitrum\|optimism\|polygon` (default `base`). Le `X-Payment` reste le tx_hash raw (ou payload x402-v2 signé pour Base). | Pas de base64 magique, header explicite, débugable avec `curl -v`. |
| D4 | Price map | Prix identiques sur les 4 chaînes (0.001 USDC single / 0.005 batch). Pas de surcharge gas côté serveur. | Gas = coût agent-side. Prix server-side = service only. Simplicité du pricing = argument marketing. |
| D5 | Replay protection | Colonne `chain` ajoutée à `x402_txs` (info uniquement). `tx_hash` reste PK unique global. | Collision tx_hash inter-chaîne = 1 / 2^256. Zéro valeur pratique à complexifier la PK. |
| D6 | 402 accepts | Challenge 402 retourne **N entries** (1 par chaîne où la treasury est configurée). Client choisit. | Spec x402-v2 compatible (`accepts` est un tableau), agents multi-chain choisissent leur préférence sans round-trip supplémentaire. |
| D7 | RPC fallback | Étendre `CHAIN_RPC_URLS` (V1.1) à Optimism + Polygon. 3 RPCs par chaîne (env primary + 2 publics). | Même stratégie de résilience que V1.1. Publics gratuits suffisent pour read-only receipts. |
| D8 | Code architecture | Créer `x402/multichain_verifier.py` (générique EVM). `base_verifier.py` devient **thin wrapper** re-exportant les fonctions pour compat imports (`x402_verify_payment_base`, `build_x402_challenge_base`). | Logique EVM = identique chain-to-chain (même ERC-20 Transfer event, même RPC `eth_getTransactionReceipt`). Factorisation propre. |

## 3. Scope backend

### 3.1 `core/config.py` — `X402_CHAIN_CONFIG`

```python
X402_SUPPORTED_CHAINS: Final[tuple[str, ...]] = (
    "base", "arbitrum", "optimism", "polygon"
)

_X402_TREASURY_DEFAULT: Final[str] = (
    os.getenv("X402_TREASURY_ADDRESS", "").strip()
    or _treasury_base_raw  # fallback to X402_TREASURY_ADDRESS_BASE (V1.0 compat)
)

def _resolve_treasury(chain: str) -> str:
    per_chain = os.getenv(f"X402_TREASURY_ADDRESS_{chain.upper()}", "").strip()
    return per_chain or _X402_TREASURY_DEFAULT

X402_CHAIN_CONFIG: Final[dict[str, dict[str, Any]]] = {
    "base": {
        "chain_id": 8453,
        "usdc_contract": "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
        "treasury": _resolve_treasury("base"),
        "network_label": "base-mainnet",
        "facilitator": X402_FACILITATOR_URL,  # Coinbase
    },
    "arbitrum": {
        "chain_id": 42161,
        "usdc_contract": "0xaf88d065e77c8cC2239327C5EDb3A432268e5831",
        "treasury": _resolve_treasury("arbitrum"),
        "network_label": "arbitrum-mainnet",
        "facilitator": None,  # direct on-chain
    },
    "optimism": {
        "chain_id": 10,
        "usdc_contract": "0x0b2C639c533813f4Aa9D7837CAf62653d097Ff85",
        "treasury": _resolve_treasury("optimism"),
        "network_label": "optimism-mainnet",
        "facilitator": None,
    },
    "polygon": {
        "chain_id": 137,
        "usdc_contract": "0x3c499c542cEF5E3811e1192ce70d8cC03d5c3359",
        "treasury": _resolve_treasury("polygon"),
        "network_label": "polygon-mainnet",
        "facilitator": None,
    },
}
```

**Validation startup** : si `ENV=prod` et **zéro** treasury n'est configurée (ni `X402_TREASURY_ADDRESS`, ni `_BASE`, ni per-chain), lever une `RuntimeError`. Si une seule est configurée, les autres chaînes sont simplement omises du 402 accepts array (pas d'erreur — l'opérateur a choisi de ne pas supporter ces chaînes).

Chaque treasury passée doit matcher le regex EVM `^0x[a-fA-F0-9]{40}$`, sinon `RuntimeError` au boot.

### 3.3 `x402/multichain_verifier.py` (nouveau, ~400 lignes)

Fonctions principales :

```python
async def verify_usdc_transfer(
    chain: str,
    tx_hash: str,
    expected_amount_raw: int | None = None,
    expected_recipient: str | None = None,
) -> dict[str, Any]:
    """Generic EVM USDC transfer receipt check. Works on any of the 4 chains."""

async def x402_verify_payment(
    chain: str, payment_header: str, expected_amount_usdc: float
) -> dict[str, Any]:
    """Dispatch: Base goes through Coinbase facilitator first then on-chain
    fallback. Other chains go direct on-chain only."""

def build_x402_challenge(
    chain: str, path: str, price_usdc: float
) -> dict[str, Any]:
    """Build one accepts entry for `chain`. Returns None if treasury
    unconfigured (caller drops the entry)."""

def build_all_accepts(path: str, price_usdc: float) -> list[dict[str, Any]]:
    """Build the list of every configured accepts entry across 4 chains."""
```

Réutilise `CHAIN_RPC_URLS` (V1.1) pour la RPC fallback pool. L'event topic ERC-20 Transfer (`0xddf252...`), les décimals USDC (6), la signature de `eth_getTransactionReceipt` sont **identiques** sur les 4 chaînes — d'où la factorisation générique.

### 3.5 `core/db.py` — migration `x402_txs`

Migration non-destructive au `init_db()` :

```python
def _ensure_x402_chain_column(conn):
    cols = {row[1] for row in conn.execute("PRAGMA table_info(x402_txs)")}
    if "chain" not in cols:
        conn.execute(
            "ALTER TABLE x402_txs ADD COLUMN chain TEXT NOT NULL DEFAULT 'base'"
        )
```

`tx_hash` reste PK (collision inter-chain = 1/2^256, cf. D5).

Mettre à jour `x402_record_tx(conn, tx_hash, amount_usdc, path, chain="base")` et `x402_tx_already_processed(conn, tx_hash)` pour sauvegarder `chain` quand fourni (default `base` pour rétrocompat).

def _parse_payment_network(request: Request) -> str:
    raw = (request.headers.get("X-Payment-Network") or "base").strip().lower()
    if raw not in X402_SUPPORTED_CHAINS:
        return "base"  # défault tolérant, verifier rejettera si treasury absente
    return raw

async def x402_middleware(request, call_next):

## 6. Non-goals V1.2 (explicites)

- ❌ Pas de Solana x402 custom (V2, conditionnel, cf. plan §5.1).
- ❌ Pas d'Ethereum x402 (gas trop cher, ~$1-5 par micropaiement de 0.001 USDC).
- ❌ Pas de Arbitrum Nova, Optimism-derivatives (Mantle, Blast), BNB, Avalanche : trop de complexité de maintenance pour peu d'ajout marketing.
- ❌ Pas de bridged USDC (`USDC.e` sur Polygon par ex.) — seulement USDC natif.
- ❌ Pas de rate-limit différencié par chaîne (on ajouterait ça si gas d'une chaîne devenait notable).
- ❌ Pas d'agrégation cross-chain de la treasury (chaque chaîne garde ses fonds, withdraw manuel).

## 7. Variables d'environnement (cheatsheet déploiement)

**V1.0 (rétrocompat)** — une seule adresse treasury, uniquement Base :
```bash
X402_TREASURY_ADDRESS_BASE=0xb3C5...  # V1.0 style, Base only
```
Comportement : seul Base dans `accepts`, 1 entry.

**V1.2 minimal** — une seule adresse, 4 chaînes :
```bash
X402_TREASURY_ADDRESS=0xb3C5...       # nouveau nom générique V1.2
```
Comportement : les 4 chaînes utilisent cette adresse, `accepts` = 4 entries.

**V1.2 avancé** — adresses séparées par chaîne (audit-friendly) :
```bash
X402_TREASURY_ADDRESS_BASE=0xAAAA...
X402_TREASURY_ADDRESS_ARBITRUM=0xBBBB...
X402_TREASURY_ADDRESS_OPTIMISM=0xCCCC...
X402_TREASURY_ADDRESS_POLYGON=0xDDDD...
```
Comportement : `accepts` = 4 entries avec chacune son `payTo`.

**Panachage** — base + arbitrum uniquement :
```bash
X402_TREASURY_ADDRESS_BASE=0xAAAA...
X402_TREASURY_ADDRESS_ARBITRUM=0xBBBB...
# optimism + polygon non configurés → absents du accepts array
```

**RPC URLs optionnels** (fallback public si absent) :
```bash
BASE_RPC_URL=https://base-mainnet.alchemyapi.io/v2/...
ARBITRUM_RPC_URL=https://arb-mainnet.g.alchemy.com/v2/...
OPTIMISM_RPC_URL=https://opt-mainnet.g.alchemy.com/v2/...
POLYGON_RPC_URL=https://polygon-mainnet.g.alchemy.com/v2/...
```
