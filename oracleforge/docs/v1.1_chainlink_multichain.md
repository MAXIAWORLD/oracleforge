# MAXIA Oracle — V1.1 : Chainlink multi-chain (lecture)

**Status** : plan figé (pré-implémentation).
**Target** : étendre `GET /api/chainlink/{symbol}` à **Base + Ethereum + Arbitrum**, parité SDK/plugins/MCP, zéro dépendance nouvelle.
**Effort** : ~1 jour.
**Breaking changes** : aucun (rétrocompat stricte — default `chain=base`).

---

## 1. Décisions figées

| # | Sujet | Décision | Motivation |
|---|---|---|---|
| D1 | RPC | User-configured prioritaire + **pool public fallback** par chaîne. Jamais 1 RPC unique en prod. | Single-RPC = single point of failure. Pool rotatif = résilience sans clé tierce payante. |
| D2 | API shape | Query param `?chain=base\|ethereum\|arbitrum` sur route existante, default `base`. | Rétrocompat V1 stricte (4 plugins + 2 SDKs pointent déjà sur l'ancienne surface). |
| D3 | Coverage | Tous feeds Chainlink officiels **crypto+forex** des 3 chaînes. Hors scope : synthetix, LP tokens, rate feeds (cross-chain, risk, reserves). | Un produit pro se différencie par la coverage (~120 symbols vs ~20). Les feeds officiels sont stables. |
| D4 | Default chain | `base` (rétrocompat V1 stricte). | V0.1.0 déjà en prod, packages publiés pointent sur cette surface. |
| D5 | `/api/price/` | **Intouché**. Reste multi-source Base uniquement (Chainlink Base + Pyth + aggregator). | L'agrégation multi-chain n'a pas de sens sémantique (même symbol peut diverger entre chaînes — c'est un signal, pas une agrégation). |

**Filtre d'acceptation par feed** (décidé ici) :
1. Description on-chain via `description()` matche le pair attendu.
2. `latestAnswer()` retourne une valeur non-nulle et `age_s < 3600`.
3. Feed listé sur https://docs.chain.link/data-feeds/price-feeds/addresses pour la chaîne cible.

Un feed qui rate le check startup est **drop** du dict (pas d'erreur 500 au boot). Log WARNING avec address + chain.

### 2.3 `_eth_call` — multi-chain avec fallback

```python
async def _eth_call(to: str, data: str, chain: str) -> str:
    rpcs = CHAIN_RPC_URLS.get(chain)
    if not rpcs:
        raise ValueError(f"unsupported chain: {chain}")
    last_exc = None
    for rpc_url in rpcs:
        try:
            return await _try_rpc(rpc_url, to, data)
        except Exception as exc:
            last_exc = exc
            continue
    raise RuntimeError(f"all {chain} RPCs failed: {last_exc!r}")
```

Comportement : rotation au premier fail (timeout, non-200, error JSON-RPC). Max 3 RPC tentés avant `raise`.

def chainlink_onchain(self, symbol: str, chain: str = "base") -> dict[str, Any]:
    """Fetch a Chainlink on-chain feed for `symbol` on `chain` (base|ethereum|arbitrum)."""
